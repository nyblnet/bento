# [Bento — 1 ファイルに収まるオフィススイート](https://bento.page/)

[English](README.md) · [简体中文](README.zh-CN.md) · 日本語

**この PowerPoint 代替アプリは、1 つの HTML ファイルです。** Bento のデッキには、
ビューアー、プレゼンター、エディターが文書内に組み込まれています。どのブラウザーでも
開いて、編集し、発表し、送信できます。受け取る側には何も必要ありません。ファイル自体が
ソフトウェアです。

**10 秒で試す：**[bento.page/slides](https://bento.page/slides) を開いてください。
これがアプリ全体であり、機能ツアーを兼ねたスターターデッキが動作します。または、
[ギャラリー](https://bento.page/) からデザイン済みテンプレートを入手して、自由に
カスタマイズできます。

**アプリをダウンロード：**[GitHub Releases](https://github.com/nyblnet/bento/releases)
ページから単一の `Bento_Slides.bento.html` を取得するか、
[bento.page](https://bento.page/releases/slides/Bento_Slides.bento.html) から直接
ダウンロードしてください（約 560 KB、アカウントもインストーラーも不要）。最新の
ブラウザーで開けば、それ自体がエディターになります。保存すると、デッキを内部に含む
状態でファイル自身を書き換えます。

## Bento が存在する理由

かつてオフィス文書は、自分が実際に*所有する*ものでした。今では、誰かのクラウドに
閉じ込められ、誰かのログイン画面の奥にあり、企業がサーバーを稼働させている間だけ
読める、借り物になっています。Bento は別の道を選びます。

- **1 ファイルを、いつまでも。** デッキ、フォント、画像、グラフ、アニメーション、
  そして完全なエディターが一緒に移動します。2026 年のコピーも 2036 年に開けます。
- **ソースを見れば一目瞭然。** データはファイル先頭のプレーンで読みやすい JSON
  ブロックにあります。バイナリ形式も、ロックインも、形式の発掘作業もありません。
- **自分自身を保存。** 保存時にファイル自身のデータブロックを書き換えます（File
  System Access API を優先し、ダウンロードへのフォールバックも用意）。アプリを
  インストールする必要はありません。
- **ローカルファーストを実証可能。** オフラインモードを有効にすると、何も端末の外へ
  出ません。更新と共同作業は強制的に無効になり、アプリにもその状態が表示されます。

## 含まれる機能

| 機能 | 説明 |
|---|---|
| **モーフプレゼンテーション** | 同じ id を共有する要素は、スライド間で位置、サイズ、色、さらにはグラデーションまでアニメーションします。スライドを複製して配置を変えるだけで、動きが自然に出来上がります。 |
| **リアルタイム共同作業** | E2EE（AES-GCM）を使用し、鍵はサーバーではなくファイル内だけに存在します。ファイル自体が招待状で、コピーを開いた人は誰でも参加できます。オフライン編集も正確にマージされます。独自の CRDT と文字単位のテキストマージも含まれます。 |
| **中身を見ないリレー** | オプションの同期リレー（[`server/sync-worker/`](server/sync-worker/)）は暗号文だけを保存し、内容を知りません。ソースを読めば分かるとおり、たった 1 ファイルの仕組みです。 |
| **組み込みグラフ** | 棒、折れ線、円、散布図を、依存関係のない独自エンジンで描画します。発表中もツールチップやズームを利用でき、棒グラフが円グラフに変わるときにはデータもモーフします。 |
| **AI 向けの設計** | 文書はファイル内のプレーン JSON なので、エージェントは `.bento.html` ファイルを直接編集でき、チャットボットは JSON を往復処理できます（`window.bento.loadDoc`）。[docs/agents.md](docs/agents.md) を参照してください。 |
| **署名付き自動更新** | リリースは ECDSA で署名され、アプリ内から提供されます。更新すると*新しい*ファイルが作成され、古いファイルはロールバック用に残ります。サーバーが文書に触れることはありません。 |
| **そのほかすべて** | スピーカービュー、コメント、レイアウト、非表示のインタラクティブ状態、ホバー表示、モーションパス、PDF エクスポート、ページサイズ、8 つの UI 言語を、約 560 KB のシェルに収めています。 |

## AI と一緒に使う

文書はファイル先頭付近の 1 つのプレーンテキストブロックに入った通常の JSON です。
そのため、ファイルを読み書きできるアシスタントなら、プラグインも API も使わずに
デッキを編集できます。方法は 2 つあります。

- **ファイル操作型ツール**は `#bento-doc` JSON を直接編集します。
  [Claude Code](https://claude.com/claude-code)、Cursor、Aider、またはファイル
  システムへアクセスできる任意のエージェントを利用できます。Claude Code ユーザーは、
  パッケージ化された `bento-slides` Skill も利用できます（このリポジトリのプラグイン
  マーケットプレイスから `/plugin marketplace
  add nyblnet/bento` でインストール）。この Skill は最新の Bento アプリも自動で
  ダウンロードします。
- **チャットでの往復処理**は、どのチャットボットでも使えます。文書の JSON をコピー
  （*About → Copy document JSON*）し、アシスタントに書き換えさせてから貼り戻します。

**ローカルのオープンウェイトモデルでも完全にオフラインで動作します。**
[Ollama](https://ollama.com)、llama.cpp、LM Studio にデッキを渡しても、何も端末の外へ
出ません。エージェントガイドは 1 ページだけなので、どのモデルのコンテキストにも
そのまま入れられます：[bento.page/agents.md](https://bento.page/agents.md)
（リポジトリ内の [docs/agents.md](docs/agents.md) にもあります）。

## 1 段落で分かるアーキテクチャ

`slides/src/model.ts` が JSON 文書モデルを定義し、単一のレンダラー（`render.ts`）が
エディターのキャンバス、サムネイル、プレゼンテーションモードを描画します（Reveal.js
がナビゲーションを担当し、モーフは DOM ではなくモデルから計算されます）。アニメーション
（`anim.ts`）とグラフ（`charts.ts`）は独自エンジンで、共同作業には独自の CRDT
（`sync/crdt.ts`。純粋なデータで、`scripts/test-sync.ts` により数十万回の収束
ファズテスト済み）を使います。シェルは約 560 KB に圧縮され、文書ブロックはプレーン
テキストのままなので、古いファイルや外部ツールでも常に差し替えられます。詳しくは
[docs/architecture.md](docs/architecture.md) を参照してください。

## セキュリティモデルを率直に説明

- 共同作業用の鍵は文書作成時にクライアント側で生成され、ファイル内だけに存在します。
  ファイルの所持がメンバー資格となり、「Rotate keys」でアクセスを取り消します。
- リレーから見えるのは、暗号文、接続時刻、ルームキーのハッシュです。内容、名前、構造は
  読み取れません。
- プレゼンス名は申告であって本人証明ではありません。共有鍵のルーム内では十分ですが、
  企業向けの本人確認には署名付きフレームが必要です（設計済み、未実装）。
- 更新チェックは静的なマニフェストを取得するだけで、利用者や文書に関する情報を送信
  しません。署名、ハッシュ、バージョンの単調増加はアプリ内で検証されます。
- 既知のトレードオフ：リアルタイム共同作業中の「元に戻す」はスナップショット方式のため、
  同じプロパティに対する共同編集者の並行編集を元に戻す場合があります。編集はデスクトップ
  優先です（スマートフォンでも閲覧と発表は快適です）。

## ソースからビルド

必要なのは Node 20+ と npm だけです。バックエンドを起動する必要のないシングルページ
ビルドです。

```bash
cd slides
npm install
npm run dev            # dev server (http://localhost:5173)
npm run build:single   # → dist-single/Bento_Slides.bento.html (the product)
```

`node scripts/test-sync.ts` は CRDT の収束テストを実行します。リリースはローカルで作成し、
署名鍵がメンテナーの端末から出ないようにしています。詳しくは
[docs/RELEASING.md](docs/RELEASING.md) を参照してください。

## ロードマップ

**Bento/Slides** は Bento/Suite 最初のアプリで、現在提供中の PowerPoint 代替です。
続いて **Docs**（`bento/docs`）と **Sheets**（`bento/sheets`）を、それぞれ独立した
自己完結型の `.bento.html` 配布物として提供します。現行リリースは
[bento.page](https://bento.page) にあり、署名付き更新チャネルを通じて既存のすべての
ファイルに届きます。
